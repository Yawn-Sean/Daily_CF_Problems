**提示 1：** 考虑每轮操作几次。

每一轮操作相当于把第 $i$ 小的元素放到对应的为止上。

相邻两轮操作之间，旋转操作是把 $i+1$ 移动到 $i$ 的右侧，所以我们只在乎这个操作次数的和。

我们先考虑 $i$ 和 $i+1$ 之间有多少数。可以发现等价于原数组中 $i$ 和 $i+1$ 之间有多少个大于 $i+1$ 的数，因为这件事是保持不变的。这件事可以用树状数组快速维护，因为你只需对大于 $i+1$ 的位置标记一个 $1$ ，再进行区间求和即可。无论你是顺序遍历 $i$ 还是逆序遍历 $i$ ，这个标记的修改次数都是 $\mathcal{O}(n)$ 的。

时间复杂度为 $\mathcal{O}(n\log n)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    t = II()
    outs = []
    
    for _ in range(t):
        n = II()
        p = LGMI()
        
        pos = [0] * n
        
        for i in range(n):
            pos[p[i]] = i
        
        ans = pos[0]
        fen = FenwickTree(n)
        
        for i in range(1, n):
            fen.add(pos[i], 1)
        
        for i in range(1, n):
            fen.add(pos[i], -1)
            
            cur = 0
            if pos[i - 1] < pos[i]: cur = fen.rsum(pos[i - 1], pos[i])
            else: cur = fen.rsum(pos[i - 1], n - 1) + fen.rsum(0, pos[i])
            
            ans += cur
        
        outs.append(ans)
    
    print('\n'.join(map(str, outs)))
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int t;
	cin >> t;

	while (t --) {
		int n;
		cin >> n;

		vector<int> pos(n);

		for (int i = 0; i < n; i ++) {
			int x; cin >> x;
			pos[x - 1] = i;
		}

		atcoder::fenwick_tree<int> fen(n);
		long long ans = pos[0];

		for (int i = 1; i < n; i ++) fen.add(pos[i], 1);

		for (int i = 1; i < n; i ++) {
			fen.add(pos[i], -1);
			ans += pos[i - 1] < pos[i] ? fen.sum(pos[i - 1], pos[i]) : fen.sum(pos[i - 1], n) + fen.sum(0, pos[i]);
		}

		cout << ans << '\n';
	}

	return 0;
}
```
