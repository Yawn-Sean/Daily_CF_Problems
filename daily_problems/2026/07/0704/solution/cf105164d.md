**提示 1：** 先算基准。然后去掉不合法的部分，最后考虑不同类别的三角形分别算了多少次。

先考虑分成三个线段的方案数，就是三个数的和不超过 $n$ 的方案数。

而 $1\leq x_1\lt x_1+x_2\lt x_1+x_2+x_3\leq n$ ，于是相当于在 $1\sim n$ 中选了 $3$ 个数，因此有 $C_n^3$ 种方法。

我们先排除那些无法构成三角形的。也就是最长的边大于等于剩余两边的和。枚举这个最长边 $z$ ，则剩余两边之和不超过 $\min(n-z,z)$ ，可以很容易算出方案数。注意这个最长边还有 $3$ 个位置可选，因此乘以 $3$ 。

这就是所有答案的部分，但这部分有重复计算。

等边三角形被算了 $1$ 遍，这部分有 $\lfloor n/3\rfloor$ 个。

非等边的等腰三角形被算了 $3$ 遍（最长边可能放到三个位置之一），考虑枚举腰 / 底边长度都可以计算，注意排除等边三角形。

剩余的三角形算了 $6$ 遍（ $3$ 边的排列）。

前两者好计算，因此第三者也好计算。最后将三个结果相加就行。

时间复杂度为 $\mathcal{O}(n)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    n = II()
    mod = 10 ** 9 + 7
    
    total = n * (n - 1) * (n - 2) // 6
    
    for z in range(1, n - 1):
        xy_bound = fmin(z, n - z)
        total -= 3 * (xy_bound - 1) * xy_bound // 2
    
    c1 = n // 3
    c2 = 0
    
    for z in range(1, n - 1):
        l = z // 2 + 1
        r = (n - z) // 2
        c2 += fmax(r - l + 1, 0) - (l <= z <= r)
    
    total -= c1 + 3 * c2
    print((total // 6 + c1 + c2) % mod)
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n;
	cin >> n;

	long long total = 1ll * n * (n - 1) * (n - 2) / 6;

	for (int z = 1; z < n - 1; z ++) {
		int xy_bound = min(z, n - z);
		total -= 3ll * (xy_bound - 1) * xy_bound / 2;
	}

	long long c1 = n / 3, c2 = 0;
	
	for (int z = 1; z <= n; z ++) {
		int l = z / 2 + 1, r = (n - z) / 2;
		c2 += max(r - l + 1, 0);
	}

	c2 -= c1;
	total -= c1 + 3 * c2;

	cout << (total / 6 + c1 + c2) % 1'000'000'007;

	return 0;
}
```
