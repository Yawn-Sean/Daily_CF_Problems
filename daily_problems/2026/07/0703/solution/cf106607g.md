**提示 1：** 想想简单情况。可能的边界情况。

首先， $k$ 较小时，答案就是 $k$ ，就是元素取 $0,1$ 即可。

而 $k$ 更大时，如果不考虑取值有上界， $k$ 与数组中奇数的个数奇偶性相同，因此是 $n$ 或者 $n-1$ 。

构造方式就是，先把非奇数的元素凑得尽可能大，剩余不断将最小的奇数加 $2$ 。

如果 $m$ 是奇数，这个过程可以不断进行，直到到达 $nm$ 附近。

否则，可能出现一堆 $m-1$ 下次没办法 $+2$ 的情况。

此时 $nm-k$ 不超过 $n$ 。

所以可以反向考虑问题，每个数分别取 $m-$ 多少，也就转化为了一系列数和为 $nm-k$ ，最多有多少个奇数，所以此时答案就是 $nm-k$ 。

时间复杂度为 $\mathcal{O}(1)$ 。


### 具体代码如下——

Python 做法如下——

```Python []
def main():
    t = II()
    outs = []
    
    for _ in range(t):
        n, m, k = MII()
        ans = n

        if ans % 2 != k % 2:
            ans -= 1
        ans = fmin(ans, k)
        if m % 2 == 0:
            ans = fmin(ans, n * m - k)
    
        outs.append(ans)
    
    print('\n'.join(map(str, outs)))
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int t;
	cin >> t;

	while (t --) {
		long long n, m, k;
		cin >> n >> m >> k;

		long long ans = n;

		if ((n - k) % 2) ans --;
		ans = min(ans, k);
		if (m % 2 == 0) ans = min(ans, n * m - k);

		cout << ans << '\n';
	}

	return 0;
}
```
