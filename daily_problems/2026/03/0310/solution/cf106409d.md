**提示 1：** 中间那条边更关键，考虑枚举它。

**提示 2：** 两侧不需要枚举太多，为什么？

长度为 $3$ 的路径，显然枚举中间那条边更合适，因为接下来只要从这条边出发再找两条边就行了。

而考虑枚举两端连出来的边。其实分别只需要考虑 $3$ 条边就好，更多的没必要，因为如果有一侧引出了三条边，只要对面引出了一条边，那么这一侧的三条边中总有一条引向新结点。而如果两侧都没有引出三条边的，则直接判断就行。

时间复杂度为 $\mathcal{O}(n+m)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main(): 
    n, m = MII()
    path = [[] for _ in range(n + 1)]
    
    for _ in range(m):
        u, v = MII()
        path[u].append(v)
        path[v].append(u)
    
    for u in range(n + 1):
        for v in path[u]:
            for x in path[u][:3]:
                for y in path[v][:3]:
                    if x != v and y != u and x != y:
                        print(x, u, v, y, sep='\n')
                        exit()
    
    print(-1)
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n, m;
	cin >> n >> m;

	vector<vector<int>> path(n + 1);
	for (int i = 0; i < m; i ++) {
		int u, v;
		cin >> u >> v;
		path[u].emplace_back(v);
		path[v].emplace_back(u);
	}

	for (int u = 1; u <= n; u ++) {
		for (auto &v: path[u]) {
			for (int i = 0; i < min(3, (int)path[u].size()); i ++) {
				for (int j = 0; j < min(3, (int)path[v].size()); j ++) {
					int a = path[u][i], b = path[v][j];
					if (a != v && b != u && a != b) {
						cout << a << '\n';
						cout << u << '\n';
						cout << v << '\n';
						cout << b << '\n';
						return 0;
					}
				}
			}
		}
	}

	cout << -1;

	return 0;
}
```
