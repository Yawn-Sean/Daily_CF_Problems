**提示 1：** 显然是博弈 DP 。我们需要定义什么状态？

**提示 2：** 要考虑哪些状态转移？如何加速？

这显然是一个博弈 DP ，先考虑状态设计。显然只跟当前所剩元素有关。

设 $dp[x]$ 表示剩下 $x$ 个元素的情况下，是必胜还是必负。

如果从一个状态能到达的所有状态都是必胜的，则这个状态必负，否则必胜。

而从 $x$ 能到达哪些状态呢？我们可以删掉 $\lceil x/2\rceil$ 个元素，所以可以到达 $[x-\lceil x/2\rceil,x-1]$ 这些状态。

我们要判断这个区间里是否有必负状态，可以求 DP 数组的前缀和，前缀和可以在 DP 的过程中维护，而判断所需的区间和可以用前缀和之差得到。

时间复杂度为 $\mathcal{O}(M)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main(): 
    M = 5 * 10 ** 5 + 5
    dp = [0] * M
    dp_acc = [0] * (M + 1)
    
    for i in range(1, M):
        if dp_acc[i] - dp_acc[i // 2] != i - i // 2:
            dp[i] = 1
        dp_acc[i + 1] = dp_acc[i] + dp[i]
    
    t = II()
    outs = []
    
    for _ in range(t):
        n = II()
        outs.append('mastermei' if dp[n] else 'the greatest')
    
    print('\n'.join(outs))
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int M = 5e5 + 5;

	vector<int> dp(M, 0), dp_acc(M, 0);

	for (int i = 1; i < M; i ++) {
		if (dp_acc[i] - dp_acc[i / 2] != i - i / 2) dp[i] = 1;
		dp_acc[i + 1] = dp_acc[i] + dp[i]; 
	}

	int t;
	cin >> t;

	while (t --) {
		int n;
		cin >> n;
		cout << (dp[n] ? "mastermei\n" : "the greatest\n");
	}

	return 0;
}
```
