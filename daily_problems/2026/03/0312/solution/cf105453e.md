**提示 1：** 将建发电厂理解为一个有点的结点给当前结点供电。

**提示 2：** 在提示 1 的情况下，所有结点之间相互连通。

先看提示 1，这样总共就有 $n+1$ 个结点，且第 $n+1$ 个结点到每个点 $i$ 的边权是 $A[i][i]$ ，最后要达成连通的目标。

所以很明显要求的就是最小生成树。

在稠密图中，可以用 Prim 算法，每次暴力找到最小成本能扩展的结点，总共进行 $\mathcal{O}(n)$ 轮，这样总复杂度不会带 $\log$ ，因为无需维护有序结构，具体可见下面的代码。

时间复杂度为 $\mathcal{O}(n^2)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main(): 
    n = II()
    grid = [LII() for _ in range(n)]
    
    ans = 0
    
    vis = [0] * n
    cur = [grid[i][i] for i in range(n)]
    
    for _ in range(n):
        chosen = -1
        
        for i in range(n):
            if not vis[i] and (chosen == -1 or cur[i] < cur[chosen]):
                chosen = i
        
        ans += cur[chosen]
        vis[chosen] = 1
        
        for j in range(n):
            cur[j] = fmin(cur[j], grid[chosen][j])
    
    print(ans)
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n;
	cin >> n;

	vector<vector<int>> grid(n, vector<int>(n));
	for (auto &x: grid) for (auto &y: x) cin >> y;

	vector<int> cur(n), vis(n, 0);
	for (int i = 0; i < n; i ++) cur[i] = grid[i][i];

	int ans = 0;

	for (int i = 0; i < n; i ++) {
		int chosen = -1;
		for (int j = 0; j < n; j ++) {
			if (!vis[j] && (chosen == -1 || cur[j] < cur[chosen])) {
				chosen = j;
			}
		}

		ans += cur[chosen];
		vis[chosen] = 1;

		for (int j = 0; j < n; j ++) {
			cur[j] = min(cur[j], grid[chosen][j]);
		}
	}

	cout << ans;

	return 0;
}
```
