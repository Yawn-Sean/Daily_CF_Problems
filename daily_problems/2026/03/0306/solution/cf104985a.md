**提示 1：** 预计越早结束的任务实际上也越早结束。

**提示 2：** 两个任务结束之间进行时间间隔的计算。

实际上，预计越早结束的任务实际上也越早结束。

为什么呢？因为一旦一个任务结束了，其余所有时间都相应进行等比例的缩短，这不会导致两个任务之间的排序关系发生变化。

于是考虑结束时间相邻的两个任务。其间的总工作量是原预计时间间隔乘以原下载速度，而新下载速度是总速度，两者相除就是这段的时间。将这些事件累计即得答案。

时间复杂度为 $\mathcal{O}(n\log n)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main(): 
    n = II()
    vs = []
    ts = []
    
    for _ in range(n):
        v, t = MII()
        vs.append(v)
        ts.append(t)
    
    st_range = sorted(range(n), key=lambda x: ts[x])
    total_v = sum(vs)
    
    res = 0
    
    cur_t = 0
    cur_v = total_v
    
    ans = [0] * n
    
    for i in st_range:
        res += (ts[i] - cur_t) * cur_v / total_v
        cur_t = ts[i]
        cur_v -= vs[i]
        ans[i] = res
    
    print('\n'.join(map(str, ans)))
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n;
	cin >> n;

	vector<long long> vs(n), ts(n);
	long long total_v = 0;

	for (int i = 0; i < n; i ++) {
		cin >> vs[i] >> ts[i];
		total_v += vs[i];
	}

	vector<int> order(n);
	iota(order.begin(), order.end(), 0);
	sort(order.begin(), order.end(), [&] (int i, int j) {return ts[i] < ts[j];});

	long long cur_v = total_v, cur_t = 0;
	vector<long double> ans(n);
	long double res = 0;

	for (auto &id: order) {
		res += (long double)(ts[id] - cur_t) * cur_v / total_v;
		ans[id] = res;
		cur_t = ts[id];
		cur_v -= vs[id];
	}

	cout << fixed << setprecision(15);
	for (auto &x: ans) cout << x << '\n';

	return 0;
}
```
