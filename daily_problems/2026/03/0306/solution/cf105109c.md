**提示 1：** 考虑以 $i$ 为最大值的子数组。

**提示 2：** 考虑一个相对暴力的做法，其实复杂度是对的。

本题可以使用笛卡尔树等方法达到更通用的做法，下面介绍一个单 $\log$ 的做法。

我们考虑枚举 $i$ ，计算以 $i$ 为最大值的满足条件的子数组个数。

我们暴力扩展左端点，直到无法扩展。接下来枚举左端点，再用双指针不断移动右端点，这样就能统计总的子数组个数。

这样做为啥复杂度是对的呢？每个位置只会被不同的 $i$ 作为左端点 / 右端点遍历 $\log$ 次，因为这些 $i$ 之间基本上至少是 $2$ 倍关系了，而数值有上限 $M$ 。

具体而言，假设从左遍历到 $i$ 这个序列是 $[j_1,j_2,j_3,\dots,j_k]$ ，则 $nums[j_{v}]\gt nums[j_{v+1}]+\dots+nums[j_k]$ 。所以即使最后两项都只有 $1$ ，更前面的项也是指数级别增长，不超过 $M$ 的项一定不超过对数个。

右侧的逻辑也是类似的。所以双指针本质上每个位置都只经过 $\log$ 次，也就没问题了。

时间复杂度为 $\mathcal{O}(n\log M)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main(): 
    t = II()
    outs = []
    
    for _ in range(t):
        n = II()
        nums = LII()
        ans = 0
        
        for i in range(n):
            cur = 0
            l = i
            while l and cur + nums[l - 1] < nums[i]:
                l -= 1
                cur += nums[l]
            
            r = i + 1
            while l <= i:
                while r < n and cur + nums[r] < nums[i]:
                    cur += nums[r]
                    r += 1
                ans += r - i
                cur -= nums[l]
                l += 1
        
        outs.append(ans)
    
    print('\n'.join(map(str, outs)))
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int t;
	cin >> t;

	while (t --) {
		int n;
		cin >> n;

		vector<int> nums(n);
		for (auto &v: nums) cin >> v;

		long long ans = 0;
		for (int i = 0; i < n; i ++) {
			int cur = 0, l = i, r = i + 1;

			while (l && cur + nums[l - 1] < nums[i]) {
				l --;
				cur += nums[l];
			}

			while (l <= i) {
				while (r < n && cur + nums[r] < nums[i]) {
					cur += nums[r];
					r ++;
				}
				ans += r - i;
				cur -= nums[l];
				l ++;
			}
		}

		cout << ans << '\n';
	}

	return 0;
}
```
