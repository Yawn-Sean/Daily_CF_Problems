**提示 1：** 考虑赋值相邻的一对，怎么样是最好的？

**提示 2：** 所以整体赋值逻辑如何？

不妨设最后数组的赋值是连续的一些正整数。

考虑最后赋值为 $v,v+1$ 的所有元素，如果将 $v,v+1$ 的赋值互换，则不影响其他的逆序对，只影响 $v,v+1$ 之间产生的逆序对。发现频率不超过 $2$ 的情况下，越早出现的元素赋越小的值是更优的。

因此从任意方案（即便是最优方案）出发，按照上述逻辑进行调整，都会使得逆序对不增加。所以直接根据最早出现的位置升序赋值即可。

时间复杂度为 $\mathcal{O}(n)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main(): 
    t = II()
    outs = []
    
    for _ in range(t):
        n = II()
        nums = LII()
        
        vis = [0] * (n + 1)
        for x in nums: vis[x] = 1
        
        mapping = [0] * (n + 1)
        pt = 1
        
        for i in range(n):
            if mapping[nums[i]] == 0:
                mapping[nums[i]] = pt
                pt += 1
            
            nums[i] = mapping[nums[i]]
        
        outs.append(' '.join(map(str, nums)))
    
    print('\n'.join(outs))
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int t;
	cin >> t;

	while (t --) {
		int n;
		cin >> n;

		vector<int> nums(n), vis(n + 1, 0);
		for (auto &x: nums) cin >> x, vis[x] = 1;

		vector<int> mapping(n + 1, 0);
		int pt = 1;
		for (auto &x: nums) {
			if (!mapping[x]) mapping[x] = pt ++;
			cout << mapping[x] << ' ';
		}
		cout << '\n';
	}

	return 0;
}
```
