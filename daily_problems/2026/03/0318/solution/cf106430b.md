**提示 1：** 考虑每一步这个数可以变成啥。

**提示 2：** 注意特判。

每一步变成的数有一个最小值和最大值的范围，这两个东西都是容易维护的。

考虑变大，最多变成两倍，而一旦这个范围超过了 $y$ ，那么显然最后一步可以用 $y$ ，因此只需看多少遍两倍操作之后可以使得 $x$ 超过 $y$ 。

考虑变小，每一次大概会变成前一次的 $\frac{2}{3}$ ，因此也只需要很少的次数就可以变小。但注意的是 $3$ 没法变小了，所以此时如果目标是 $1,2$ 直接输出不可能。

如果不变的话，显然是 $0$ 次操作。

时间复杂度为 $\mathcal{O}(\log M)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main(): 
    t = II()
    outs = []
    
    for _ in range(t):
        x, y = MII()
        
        if x == y: outs.append(0)
        elif x < y:
            ans = 0
            while x < y:
                x *= 2
                ans += 1
            outs.append(ans)
        else:
            if y <= 2: outs.append(-1)
            else:
                ans = 0
                while x > y:
                    x = 2 * x // 3 + 1
                    ans += 1
                outs.append(ans)
    
    print('\n'.join(map(str, outs)))
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int t;
	cin >> t;

	while (t --) {
		int x, y;
		cin >> x >> y;

		if (x == y) cout << "0\n";
		else if (x < y) {
			int ans = 0;
			while (x < y) {
				x *= 2;
				ans ++;
			}
			cout << ans << '\n';
		}
		else {
			if (y <= 2) cout << "-1\n";
			else {
				int ans = 0;
				while (x > y) {
					x = 2 * x / 3 + 1;
					ans ++;
				}
				cout << ans << '\n';
			}
		}
	}

	return 0;
}
```
