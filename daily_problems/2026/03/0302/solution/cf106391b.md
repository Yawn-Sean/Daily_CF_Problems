**提示 1：** 似乎完全证明不了无法构造，无从下手，所以猜可以构造。

**提示 2：** 将问题转化为更小规模的问题。

先看提示 1，我们猜总能构造。考虑数学归纳法。

在 $n=1$ 时，显然可以。假设 $n=k$ 时也可以构造，考虑 $n=k+1$ 。

此时我们只需构造出最后一条边，则去掉这条边上的一个点，剩余的 $k$ 个点根据归纳假设也可以构造，加上这条边也就得到了 $n=k+1$ 时的构造。

而最后一条边需要满足连结的结点的数值之差是 $k$ 的倍数。而总共有 $k+1$ 个数，根据抽屉原理，一定有两个数模 $k$ 同余，取这两个元素即可。寻找的过程中只需将元素关于取模结果进行分类即可。

综上得到证明和构造。

这题我曾经想出过，不过大概是让找是否有满足要求的生成树，表面上看上去可能自由度更大，但可能实际上并没有变简单。

时间复杂度为 $\mathcal{O}(n^2)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main(): 
    t = II()
    outs = []
    
    for _ in range(t):
        n = II()
        nums = LII()
        used = [0] * n
        
        outs.append('YES')
        ops = []
        
        for i in range(n - 1, 0, -1):
            vis = [-1] * i
            
            for j in range(n):
                if not used[j]:
                    x = nums[j] % i
                    if vis[x] != -1:
                        used[j] = 1
                        ops.append(f'{vis[x] + 1} {j + 1}')
                        break
                    vis[x] = j
        
        outs.append('1')
        ops.reverse()
        outs.append('\n'.join(ops))
    
    print('\n'.join(outs))
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int t;
	cin >> t;

	while (t --) {
		int n;
		cin >> n;

		vector<int> nums(n);
		for (auto &x: nums) cin >> x;

		cout << "YES\n1\n";

		vector<pair<int, int>> ops;
		vector<int> used(n, 0);

		for (int i = n - 1; i >= 1; i --) {
			vector<int> vis(i, -1);
			for (int j = 0; j < n; j ++) {
				if (!used[j]) {
					int x = nums[j] % i;
					if (vis[x] != -1) {
						ops.emplace_back(vis[x], j);
						used[j] = 1;
						break;
					}
					vis[x] = j;
				}
			}
		}

		reverse(ops.begin(), ops.end());

		for (auto &[x, y]: ops) cout << x + 1 << ' ' << y + 1 << '\n';
	}

	return 0;
}
```
