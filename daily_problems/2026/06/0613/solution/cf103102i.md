**提示 1：** $1,2$ 随意放，剩余的元素排布有啥要求？

**提示 2：** 如何解决这部分的？

先把问题看成一个环，环上相邻两点满足取模后不超过 $2$ 。

对于序列中大于 $2$ 的元素，其后面的元素只能比它小，不然取模就是它自己了，就矛盾了。

所以如果用 $1,2$ 把整个环分成两段，每一段都是一个递减序列。

所以我们相当于要构造两个递减序列，总共用的元素是 $3,\dots,n$ ，满足前一项关于后一项取模不超过 $2$ 。

把这个序列反过来，然后把相邻的数字看成一段，则段内一定满足要求。而两个段之间的相邻元素 $x,y$ 才是非常需要关心的问题。

因此一段段 DP ，也就是把 $3,4,\dots, n$ 的元素切分成一段一段的，再把相邻的段放到不同的序列中，就是整体拆序列的方案数了。

假设我们现在某一段分段的结尾时是 $i$ （这里令 $i\lt n$ 且 $i+1$ 所在的段跟前面的不产生矛盾），对应方案数有 $dp[i]$ 种，有哪些状态转移呢？

首先，可以让下一段直接走到 $n$ ，一定满足要求。

否则，假设走到了 $j$ ，则 $j+1$ 是跟这个 $i$ 被分到同一个序列的，因此有 $(j+1)\bmod i\leq 2$ 。我们枚举 $ki+b$ ( $b\leq 2$ ) 即可。

对于不同的 $i$ ，枚举的数量是 $\mathcal{O}(n/i)$ 量级的，因此求和后是个 “调和级数” 的复杂度，为 $\mathcal{O}(n\log n)$ 。

段落分完后，可以分别选择放到 $1$ 和 $2$ 后，因此这里方案数得乘 $2$ 。而 $1$ 的位置任选，因此还需要将答案乘 $n$ 。

注意 $n\lt 3$ 的时候上述讨论并不成立，需要特判。

时间复杂度为 $\mathcal{O}(n\log n)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    n = II()
    mod = 10 ** 9 + 7
    
    if n < 3:
        print(n)
    else:
        dp = [1] * (n + 1)
        
        for i in range(3, n):
            dp[n] += dp[i]
            dp[n] %= mod
            
            for j in range(2 * i - 1, n, i):
                dp[j] += dp[i]
                dp[j] %= mod
            
            for j in range(2 * i, n, i):
                dp[j] += dp[i]
                dp[j] %= mod
            
            for j in range(i + 1, n, i):
                dp[j] += dp[i]
                dp[j] %= mod
        
        print(dp[n] * 2 * n % mod)
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n, mod = 1e9 + 7;
	cin >> n;

	if (n < 3) cout << n;
	else {
		vector<int> dp(n + 1, 1);

		for (int i = 3; i < n; i ++) {
			dp[n] = (dp[n] + dp[i]) % mod;
			for (int j = 2 * i - 1; j < n; j += i)
				dp[j] = (dp[j] + dp[i]) % mod;
			for (int j = 2 * i; j < n; j += i)
				dp[j] = (dp[j] + dp[i]) % mod;
			for (int j = i + 1; j < n; j += i)
				dp[j] = (dp[j] + dp[i]) % mod;
		}

		cout << 2ll * n * dp[n] % mod;
	}

	return 0;
}
```
