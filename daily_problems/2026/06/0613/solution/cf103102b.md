**提示 2：** 一个经典博弈问题。

操作实在太多了，得想办法减少一些需要考虑的东西。

如果你考虑整体的逆序对，你会发现第一个操作让逆序对减少 $1$ ，后面的操作让逆序对减少 $2$ 。

而一旦逆序对至少有一个，则总能进行第一个操作；一旦逆序对至少有两个，则总能进行第二个操作（可以想下，不可能构造一个不包含第二类子串的至少有两个逆序对的序列）。

所以操作实际上就只是针对逆序对数量操作。而每次操作可以选择减去 $1$ 或者 $2$ ，看谁做最后一次操作。

这就是极其经典的博弈论问题了。在 $3$ 的倍数的情况下，先手选 $x$ 后手就选 $3-x$ ，后手总是必胜；否则，先手总能一次操作达到 $3$ 的倍数，操作后变成后手，于是也必胜了。

于是，只需统计逆序对关于 $3$ 取模的余数即可。

时间复杂度为 $\mathcal{O}(n)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    s = [int(c) for c in I()]
    c1 = 0
    rev = 0
    
    for x in s:
        if x: c1 = (c1 + 1) % 3
        else: rev = (rev + c1) % 3
    
    print('Alice' if rev else 'Bob')
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	string s;
	cin >> s;

	int c1 = 0, rev = 0;

	for (auto &c: s) {
		if (c == '1') c1 = (c1 + 1) % 3;
		else rev = (rev + c1) % 3;
	}

	cout << (rev ? "Alice" : "Bob");

	return 0;
}
```
