**提示 1：** 本质上是求最大流。用最小割求即可。

看提示即可。而整体最小割相当于每个部分的最小割。

这里最小割可能是 $a+d,b+c,a+c+e,b+c+d$ 之一，取最小值，再对所有部分取最小值即可。

当然你也可以仔细分析每一块到底可以通行多少，结论跟上述描述是一致的。

时间复杂度为 $\mathcal{O}(n)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    n = II()
    ans = 10 ** 9
    
    for _ in range(n):
        a, b, c, d, e = MII()
        ans = fmin(ans, min(a + d, b + e, a + c + e, b + c + d))
    
    print(ans)
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n, ans = 1e9;
	cin >> n;

	while (n --) {
		int a, b, c, d, e;
		cin >> a >> b >> c >> d >> e;
		ans = min(ans, min({a + d, b + e, a + c + e, b + c + d}));
	}

	cout << ans;

	return 0;
}
```
