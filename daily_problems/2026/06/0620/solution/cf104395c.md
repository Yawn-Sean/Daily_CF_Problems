**提示 1：** 一些性质：如果我选择了按单独第一个字符 $k$ 次，意味着什么？

**提示 2：** 不同操作间可以任意排序。

如果我们选择了第一个字符单独取 $a$ 次，跟第二个字符一起取的次数就是 $k-a$ 次，以此类推，所有取的子字符串都完全确定。

而最后的拼接就是把所有子字符串排序后再拼接。

发现：如果单独取第一个字符得到的分块结果比取前两个字符的结果好，那我不如都按照第一个来拆；否则按照第二个来拆。

也就是只需考虑：

$s_1+s_2s_3+s_4s_5+\dots$

和

$s_1s_2+s_3s_4+s_5s_6+\dots$

两种拆法。

拼接的时候就是经典的 $a+b\lt b+a$ 的判断。但这里可以直接根据字典序排序，因为没有重复字符。

时间复杂度为 $\mathcal{O}(nk)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    n, k = MII()
    s = I()
    
    ans = 'a'
    
    for i in range(2):
        tmp = []
        
        for j in range(i - 2, n, 2):
            tmp.append(s[fmax(j, 0):j + 2])
        
        tmp.sort()
        ans = min(ans, ''.join(x * k for x in tmp))
    
    print(ans)
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n, k;
	cin >> n >> k;

	string s;
	cin >> s;

	string ans = "a";

	for (int i = 0; i < 2; i ++) {
		vector<string> tmp;
		if (i) tmp.emplace_back(s.substr(0, 1));

		for (int j = i; j < n; j += 2)
			tmp.emplace_back(s.substr(j, min(2, n - j)));

		sort(tmp.begin(), tmp.end());

		string tmp_ans;

		for (auto &x: tmp) {
			for (int i = 0; i < k; i ++) {
				tmp_ans += x;
			}
		}

		ans = min(ans, tmp_ans);
	}

	cout << ans;

	return 0;
}
```
