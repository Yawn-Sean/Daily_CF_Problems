**提示 2：** 考虑一个图不断加点，环数如何变化？

首先把所有原图中的环处理掉，加入答案。

接下来，其实可以把每一条链看成一个点，不影响答案。

这就等价一个排列，其视为一个图后，期望的环数。

考虑大小为 $k$ 的图的答案推到大小为 $k+1$ 的图。

此时新增一个点，这个点可以自成一环，概率为 $\frac{1}{k+1}$ ，其余情况下，可以成为原有点的一个后继，然后再连回这个环的下一个点。

所以期望增加环数就是 $\frac{1}{k+1}$ 。

而这里应该有 $n-m$ 个点，所以答案就是 $1+\frac{1}{2}+\frac{1}{3}+\dots+\frac{1}{n-m}$ 。

时间复杂度为 $\mathcal{O}(n+m)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    n, m = MII()
    uf = UnionFind(n)
    
    mod = 10 ** 9 + 7
    f = Factorial(n, mod)
    
    ans = 0
    
    for _ in range(m):
        u, v = GMI()
        if not uf.merge(u, v):
            ans += 1
    
    for i in range(1, n - m + 1):
        ans += f.inv(i)
        ans %= mod
    
    print(ans)
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n, m, mod = 1e9 + 7;
	cin >> n >> m;

	atcoder::dsu uf(n);
	int ans = 0;

	for (int i = 0; i < m; i ++) {
		int u, v;
		cin >> u >> v;
		u --, v --;
		if (!uf.merge(u, v)) ans ++;
	}

	for (int i = 1; i <= n - m; i ++) {
		ans += quickPow(i, mod - 2, mod);
		ans %= mod;
	}

	cout << ans;

	return 0;
}
```
