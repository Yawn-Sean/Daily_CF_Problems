**提示 1：** 先列出所谓的 “转移方程” 。

**提示 2：** 我们实际上要找整体不动点。

不妨设到第 $i$ 个位置时，剩余时间 + 距离的最优期望是 $f(i)$ 。

则考虑当前一期是否有空位，以及有空位的情况下的决策，应该满足：

$f(i)=p_i\min(f(i+1)+1,d_i)+(1-p_i)(f(i+1)+1)$ 

这个逻辑很类似于 DP，但我们没有初值，所以没办法用 DP 解这件事。

不妨设 $f_i(x)=p_i\min(x+1,d_i)+(1-p)(x+1)$ ，则本题的答案 $x$ 应该是方程 $x=f_1(f_2(\dots (f_n(x)) \dots))$ 的解。

设 $f(x)=f_1(f_2(\dots (f_n(x)) \dots))$ ，则因为 $f_i$ 对于每个 $x$ 而言，每一段的斜率都在 $(0,1]$ 之间，所以 $f$ 对于每个 $x$ 而言，斜率也都在 $(0,1]$ 之间。

同时，如果 $x$ 足够小，整个函数就是 $x+n$ ，且 $f(x)$ 过了这段之后，每一段的斜率严格小于 $1$ 。所以总能找到解。

同时 $f(x)-x$ 单调不增，且保持不变的一段不是解，所以 $f(x)-x$ 存在唯一零点。

因为整体不增，所以可以二分求 $f(x)-x=0$ 零点。整体递减，所以如果 $f(x)-x\lt 0$ ，则 $r=mid$ ，否则 $l=mid$ 。

需要担心数值过大的问题吗？答案是否定的，因为每一轮的结果不超过 $\max(x+1,d_i)$ 。

二分上界用多少？最坏的情况情况就是所有的 $p_i=0.001,d_i=10^6$ ，这样，期望为 $\frac{1}{p_i}-1+10^6$ （因为第一步不用走），所以非常可控。

时间复杂度为 $\mathcal{O}(n\log\epsilon^{-1})$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    n = II()
    ds = LII()
    probs = LFI()
    
    l, r = 0, 10 ** 9
    
    for _ in range(100):
        mid = (l + r) / 2
        cur = mid
        
        for i in range(n - 1, -1, -1):
            cur = probs[i] * fmin(cur + 1, ds[i]) + (1 - probs[i]) * (cur + 1)
    
        if cur < mid: r = mid
        else: l = mid
    
    print((l + r) / 2)
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n;
	cin >> n;

	vector<int> ds(n);
	for (auto &x: ds) cin >> x;

	vector<long double> ps(n);
	for (auto &x: ps) cin >> x;

	long double l = 0, r = 1e9;

	for (int i = 0; i < 100; i ++) {
		long double mid = (l + r) / 2, cur = mid;

		for (int j = n - 1; j >= 0; j --) {
			cur = ps[j] * min(cur + 1, (long double)ds[j]) + (1 - ps[j]) * (cur + 1);
		}

		if (cur < mid) r = mid;
		else l = mid;
	}

	cout << fixed << setprecision(15) << (l + r) / 2;

	return 0;
}
```
