**提示 1：** $1\sim n$ 只能做一对中更小的那个数，为什么？

**提示 2：** 分别考虑 $1\sim n$ 之间的数有多少种方案，从限制强的考虑到限制弱的。

先看提示 1 ，为啥呢？

如果有一个 $1\sim n$ 中的数在一对数中作为大数出现，那么一定有一个 $n+1\sim 2n$ 中的数作为小数出现（大数总共只有 $n$ 个），而这件事是不可能的。

所以一定是小的数是 $1\sim n$ 中取，大的数是 $n+1\sim 2n$ 中取。

接下来分别考虑 $1\sim n$ 分别和谁配对。因为跟更大的数能配对的一定能跟更小的数配对，所以我们从限制更强的开始。

假设从大到小考虑到 $i$ 。 $i$ 只能跟 $\max(n,2i)\sim n$ 之间的数搭配，而其中 $i+1\sim n$ 已经用掉了 $n-i$ 个数，剩余的数可以任意选择。可以发现这个数的选择方案数量跟之前的选择方案无关。

所以将这些数量相乘就是答案。

时间复杂度为 $\mathcal{O}(n)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    n = II()
    ans = 1
    
    for i in range(1, n + 1):
        ans *= fmin(i, n - i + 1)
    
    print(ans)
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n;
	cin >> n;

	long long ans = 1;
	for (int i = 1; i <= n; i ++)
		ans = ans * min(i, n - i + 1);

	cout << ans;

	return 0;
}
```
