**提示 1：** 仔细分析——最终的集合需要满足什么条件？

首先，不能有两个集合的中位数都是负数，不然把它们合并到一起，中位数在原先的两个负数之间，一定比它们的和要大。

接下来考虑中位数是非负数的集合。

首先，这些集合不能有偶数个元素，否则考虑中间的两个元素 $A,B$ ，把其中一个取出单独形成一个集合，则原先中位数是 $\frac{A+B}{2}$ ，后来中位数变成 $A+B$ ，变大了。

而如果有两个大小不小于 $3$ 的中位数是非负数的集合，我们可以把其中中位数较大的集合的最大数与最小数删掉，加到另一个集合中，这样整体中位数之和不会变小。不断进行操作，可以将其中一个大小变为 $1$ 。

综上，最后只可能由一系列非负数的单元素集合，配上一个大小为奇数的中位数非负的集合，再加上一个中位数为负数的集合组成。

事实上，如果既存在一个大小为奇数的中位数非负的集合，又存在一个中位数为负数的集合，我们可以利用类似两段前的操作，将非负集合的最小值和最大值添加到负数集合去，使得整体中位数和不变小，如此操作，直到非负数集合大小也变成 $1$ 。

于是，最后答案就只可能是一系列大小为 $1$ 的非负数单元素集合，配上另一个集合 $S$ 。

$S$ 中首先一定包含所有的负数。

如果 $S$ 的中位数是负数，那么加入到 $S$ 中的非负数一定越小越好（因为只要是非负数，在 $S$ 中都起到大于中位数的作用，都一样）。

否则，如果有 $K$ 个负数，则这个集合大小一定是 $2K+1$ 。首先大小至少是 $2K+1$ 且为奇数，同时如果大小不小于 $2K+3$ ，中间的三个数都是非负数，取出其中最大的和最小的一定使得中位数和增大。

而此时选择最小的 $K+1$ 个非负数是最优的，只需考虑需要添加 $K$ 个权值是 $0$ 的位置，显然这 $K$ 个位置越小越好。

综上，一定是最小的若干个数形成一个集合，剩余数字单独成集。于是排序枚举分割点，对每个分割点计算结果即可。

时间复杂度为 $\mathcal{O}(n\log n)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    n = II()
    nums = LII()
    nums.sort()
    
    total = 2 * sum(nums)
    ans = total
    
    for i in range(n):
        total -= 2 * nums[i]
        ans = fmax(ans, total + nums[i // 2] + nums[i - i // 2])
    
    print(ans)
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n;
	cin >> n;

	vector<int> nums(n);
	long long total = 0;
	for (auto &x: nums) cin >> x, total += x * 2;

	sort(nums.begin(), nums.end());

	long long ans = total;

	for (int i = 0; i < n; i ++) {
		total -= nums[i] * 2;
		ans = max(ans, total + nums[i / 2] + nums[i - i / 2]);
	}

	cout << ans;

	return 0;
}
```
