**提示 1：** 显然 bitmask DP。过程中如何考虑逆序对和 $|i-p[i]|$ 的和？

**提示 2：** 如何让一个 DP 维度限制在 $0\sim k$ ？

如果我们目前这个数组前缀是 $msk$ ，此时来了个新数 $x$ 连着前缀，则这个新数和后面所有数形成的逆序对数是很容易确定的（即后续的数中比当前数小的数）。

而 $p$ 是排列，所以 $i-p[i]$ 求和为 $0$ ，题中的求和式等价于 $\max(i-p[i],0)$ 的和或者 $\max(p[i]-i,0)$ 的和。

所以考虑当前选取数集合为 $msk$ 时，逆序对和 $\max(p[i]-i,0)$ 的和的差为 $i$ 时的方案数量为 $dp[i][msk]$ 。

值得注意的是这两者之差始终非负。因为 $\max(p[i]-i,0)$ 的和是以 $i$ 为逆序对左端点的逆序对数量的一个更小的估计。

这样我们就保证了 DP 的一个维度是在 $[0,k]$ 之间的，总状态数也就被限制住了（虽然好像你写个 $\mathcal{O}(n^2)$ 的也能过）。

时间复杂度为 $\mathcal{O}(kn2^n)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main(): 
    n, k = MII()
    mod = 10 ** 9 + 7
    
    dp = [[0] * (1 << n) for _ in range(k + 1)]
    dp[0][0] = 1
    
    for i in range(k + 1):
        for j in range(1 << n):
            cnt = j.bit_count()
            smaller = 0
            for bit in range(n):
                if j >> bit & 1: continue
                ni = i + smaller - fmax(bit - cnt, 0)
                nj = j | (1 << bit)
                if ni <= k:
                    dp[ni][nj] += dp[i][j]
                    dp[ni][nj] %= mod
                smaller += 1
    
    print(dp[k][-1])
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n, k, mod = 1e9 + 7;
	cin >> n >> k;

	vector<vector<int>> dp(k + 1, vector<int>(1 << n, 0));
	dp[0][0] = 1;

	for (int i = 0; i <= k; i ++) {
		for (int j = 0; j < (1 << n); j ++) {
			int cnt = __popcount(j);
			int smaller = 0;
			for (int bit = 0; bit < n; bit ++) {
				if (j >> bit & 1) continue;
				int ni = i + smaller - max(bit - cnt, 0);
				int nj = j | (1 << bit);
				if (ni <= k)
					dp[ni][nj] = (dp[i][j] + dp[ni][nj]) % mod;
				smaller ++;
			}
		}
	}

	cout << dp[k].back();

	return 0;
}
```
