**提示 1：** 如果操作了，那么最后相同的数值如何确定？

**提示 2：** 知道最后数值了的话，最少操作几次？

如果数组本身就满足条件，显然操作次数等于 $0$ 。

否则，最后的数组的或和只可能是原先的或和与 $x$ 的或，设为 $V$ 。

接下来考虑何时无法达成目标。我们强行让每个元素都和 $x$ 取或，如果这样都不行，那肯定无论怎么操作都不行。

此时一定存在某一位，使得这一位在原数组中为 $0$ ，但数组整体的或和在该位非 $0$ ，且 $x$ 无法改变这一点。于是我们可以先统计那些在 $x$ 中不得不是 $1$ 的位，以方便后续判断。

这部分怎么处理呢？我们可以用位运算，提取每个元素需要变成 $1$ 的位，将这些结果或起来就是必须是 $1$ 的位了。

能达成目标时，分两种情况——

- 如果存在元素无需操作，则 $x$ 操作不影响这些元素，也就是 $x$ 是原本数组或和的子集。这些情况是等价的，可以开头预处理。为此我们贪心做就行，依次遍历所有需要操作的位置，一旦前一次操作无法覆盖当前位置，我们新开一次操作，直到遍历到数组尾端。

- 否则所有元素都得操作。操作次数等于 $\lceil n/m\rceil$ 。

时间复杂度为 $\mathcal{O}(n+q)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main(): 
    t = II()
    outs = []
    
    for _ in range(t):
        n, m = MII()
        nums = LII()
        
        total_or = reduce(ior, nums)
        op_must = 0
        
        ans = 0
        cur = -1
        
        for i in range(n):
            if nums[i] != total_or:
                op_must |= total_or - nums[i]
                if i > cur:
                    ans += 1
                    cur = i + m - 1
        
        q = II()
        for _ in range(q):
            x = II()
            if x & op_must != op_must: outs.append(-1)
            elif x & total_or == x or ans == 0: outs.append(ans)
            else: outs.append((n - 1) // m + 1)
    
    print('\n'.join(map(str, outs)))
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int t;
	cin >> t;

	while (t --) {
		int n, m;
		cin >> n >> m;

		vector<int> nums(n);
		int total_or = 0;
		for (auto &x: nums) cin >> x, total_or |= x;

		int op_must = 0, ans = 0, cur = -1;
		for (int i = 0; i < n; i ++) {
			if (nums[i] != total_or) {
				op_must |= total_or ^ nums[i];
				if (i > cur) {
					ans ++;
					cur = i + m - 1;
				}
			}
		}

		int q;
		cin >> q;

		while (q --) {
			int x;
			cin >> x;

			if ((x & op_must) != op_must) cout << -1 << '\n';
			else if ((x & total_or) == x || ans == 0) cout << ans << '\n';
			else cout << (n - 1) / m + 1 << '\n';
		}
	}

	return 0;
}
```
