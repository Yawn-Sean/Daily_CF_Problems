**提示 1：** 先分析下最后集合大概需要满足什么条件？

一个大小不小于 $3$ 的集合中可能有正数吗？

不可能，因为如果有，我可以把最大的数和最小的数拎出来形成一个集合，剩余的数形成一个集合，这样结果更大。

所以每个正数要么单独是一个集合，要么跟一个负数配成一个集合。

如果正数数量足够多，就可以把负数全部都配上对，最后答案就是所有正数的和。

否则，我们可以尽量让这些正数消灭负数。

剩余的负数如果再拆一定有一部分的中位数比原中位数还小，所以剩余的负数直接单独成组。

所以应该剩下那些更大的负数，再取其中位数即可（事实上，这个中位数就是原数组的中位数）。

根据上述讨论求值即可。

时间复杂度为 $\mathcal{O}(n\log n)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    n = II()
    nums = LII()
    nums.sort()
    print(sum(fmax(0, x) for x in nums) + fmin(nums[n // 2], 0))
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n;
	cin >> n;

	vector<int> nums(n);
	for (auto &x: nums) cin >> x;
	sort(nums.begin(), nums.end());

	long long total = 0;
	for (auto &x: nums) total += max(0, x);

	cout << total + min(nums[n / 2], 0);

	return 0;
}
```
