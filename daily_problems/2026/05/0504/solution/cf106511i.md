**提示 1：** 查询列表不宜过短 / 过长，为什么？

**提示 2：** 根据提示 1，设计合适的策略。

查询列表如果太短，很容易查询到空的，不好。

查询列表如果太长，那么我们总查询次数会很受限，也不好。

考虑将两部分都分成 $B$ 块。

对于左侧的每一块 $L_0$ ，我们可以遍历右侧的每一块，不断查询——

- 如果查到了边，则左侧这个点去掉。

- 否则右侧这块没有作用了，继续下一块。

这样的总成本如何呢？

每次查询对应的块长和是 $\mathcal{O}(\frac{n}{B})$ 量级的。而对于每一个左侧的找到边的点，都有一次查询是帮助它完成这件事的；而每一个右侧的块，都需要经历一次找不到边的查询。因此总查询次数是 $\mathcal{O}(\frac{n}{B}+B)$ 。

总共需要对 $B$ 块左侧的点执行上述的事儿，因此总查询成本是 $\mathcal{O}(B\times\frac{n}{B}(\frac{n}{B}+B))=\mathcal{O}(n(\frac{n}{B}+B))$ ，在 $B=\mathcal{O}(\sqrt{n})$ 时最优。

查询次数为 $\mathcal{O}(n)$ ，总时间复杂度为 $\mathcal{O}(n^{1.5})$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    def query(v1, v2):
        if len(v1) == 0 or len(v2) == 0: return (-2, -2)
        print('?', len(v1), *(x + 1 for x in v1), len(v2), *(x + 1 for x in v2), flush=True)
        return GMI()
    
    n = II()
    ans = [-2] * n
    
    block = 20
    
    for i in range(0, n, block):
        for j in range(0, n, block):
            while True:
                to_query1 = [x for x in range(i, i + block) if x < n and ans[x] == -2]
                to_query2 = [x for x in range(j, j + block) if x < n]
                u, v = query(to_query1, to_query2)
                if u >= 0: ans[u] = v
                else: break
    
    print('!', *(x + 1 for x in ans))
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n;
	cin >> n;

	int block = 20;

	vector<int> ans(n + 1, -1);

	auto update = [&] (vector<int> &v1, vector<int> &v2) -> bool {
		if (v1.empty()) return false;
		cout << "?";
		cout << ' ' << v1.size();
		for (auto &x: v1) cout << ' ' << x;
		cout << ' ' << v2.size();
		for (auto &x: v2) cout << ' ' << x;
		cout << endl;

		int x, y;
		cin >> x >> y;

		if (x < 0) return false;
		ans[x] = y;
		return true;
	};

	for (int i = 1; i <= n; i += block) {
		for (int j = 1; j <= n; j += block) {
			while (true) {
				vector<int> to_query1, to_query2;

				for (int x = i; x < i + block && x <= n; x ++) {
					if (ans[x] == -1) {
						to_query1.emplace_back(x);
					}
				}

				for (int x = j; x < j + block && x <= n; x ++) {
					to_query2.emplace_back(x);
				}

				if (!update(to_query1, to_query2)) break;
			}
		}
	}

	cout << "!";
	for (int i = 1; i <= n; i ++) cout << ' ' << ans[i];

	return 0;
}
```
