**提示 1：** 我们相当于要最大化遇到当前第 $k$ 个元素时的 MEX 数值。尝试给出一个上界。

**提示 2：** 这个上界可以达到吗？怎么达到？

我们相当于要判断到这个元素时， MEX 能否严格大于这个元素。考虑最大化 MEX 。

这个 MEX 数值的上界是其余元素的 MEX ，能否达到呢？

答案是肯定的，我们直接将当前元素移动到末尾即可，这样我们取的元素一定包含 $0,1,\dots,mex-1$ 。

所以我们只需判断其余元素的 MEX 是否大于当前元素即可。如果满足，将当前元素移动到末尾，否则返回无解。

时间复杂度为 $\mathcal{O}(n)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    t = II()
    outs = []
    
    for _ in range(t):
        n, k = MII()
        nums = LII()
        
        nums[k - 1], nums[-1] = nums[-1], nums[k - 1]
        
        vis = [0] * (n + 1)
        for i in range(n - 1):
            vis[nums[i]] = 1
        
        mex = 0
        while vis[mex]:
            mex += 1
        
        outs.append('-1 -1' if nums[-1] >= mex else f'{k} {n}')
    
    print('\n'.join(outs))
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int t;
	cin >> t;

	while (t --) {
		int n, k;
		cin >> n >> k;

		vector<int> nums(n);
		for (auto &x: nums) cin >> x;

		swap(nums[k - 1], nums[n - 1]);

		vector<int> vis(n + 1, 0);
		for (int i = 0; i < n - 1; i ++)
			vis[nums[i]] = 1;
		
		int mex = 0;
		while (vis[mex]) mex ++;

		if (nums.back() >= mex) cout << -1 << ' ' << -1 << '\n';
		else cout << k << ' ' << n << '\n';
	}

	return 0;
}
```
