**提示 2：** 还有啥条件？满足的情况下怎么构造？

如果图中有环，可以绕着环走 $67$ 圈出来一个不合法路径。

所以图中不能有环。于是有拓扑序。

如果只有一个数码，则图中不能有 $67$ 长度的路径，否则这一条路径就矛盾了。

如果有多个数码呢？还能有长度为 $67$ 的路径吗？

答案是否定的。考虑一个长度为 $67$ 的数组，其前缀和长度为 $68$ ，因此总有两个不同位置的前缀和关于 $67$ 取模相等，于是这两个位置之间的子数组求和是 $67$ 的倍数。也就矛盾了。

所以图中的最长路径的长度不能大于等于 $67$ 。而如果这个数值小于 $67$ ，显然只用一个数码就够了。最长路径可以 DP 得到。

综上，判断有无环和长度不小于 $67$ 的路径，如果都没有，全部用同一个边权即可。

时间复杂度为 $\mathcal{O}(n+m)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():  
    n, m = MII()
    s = I()
    
    path = [[] for _ in range(n)]
    indeg = [0] * n
    
    for _ in range(m):
        u, v = GMI()
        path[u].append(v)
        indeg[v] += 1
    
    dp = [0] * n
    stk = [i for i in range(n) if indeg[i] == 0]
    
    while stk:
        u = stk.pop()
        
        for v in path[u]:
            dp[v] = fmax(dp[v], dp[u] + 1)
            indeg[v] -= 1
            if indeg[v] == 0:
                stk.append(v)
    
    if max(indeg) or max(dp) >= 67: print('NO')
    else:
        print('YES')
        print('\n'.join(s[0] for _ in range(m)))
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n, m;
	cin >> n >> m;

	string s;
	cin >> s;

	vector<vector<int>> path(n);
	vector<int> indeg(n, 0);

	for (int i = 0; i < m; i ++) {
		int u, v;
		cin >> u >> v;
		u --, v --;
		path[u].emplace_back(v);
		indeg[v] ++;
	}

	vector<int> depth(n, 0), stk;

	for (int i = 0; i < n; i ++) {
		if (!indeg[i]) {
			stk.emplace_back(i);
		}
	}

	while (!stk.empty()) {
		int u = stk.back(); stk.pop_back();

		for (auto &v: path[u]) {
			depth[v] = max(depth[v], depth[u] + 1);
			indeg[v] --;

			if (!indeg[v]) stk.emplace_back(v);
		}
	}

	if (*max_element(depth.begin(), depth.end()) >= 67 || *max_element(indeg.begin(), indeg.end()))
		cout << "NO\n";
	else {
		cout << "YES\n";
		while (m --) cout << s[0] << '\n';
	}

	return 0;
}
```
