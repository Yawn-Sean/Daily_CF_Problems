**提示 1：** 想想简单的情况。

**提示 2：** 好像有办法用数学归纳法解决剩余的情况。

首先，如果开头就都是 $k$ 的倍数，直接先手输。

如果只有两个数不是 $k$ 的倍数，操作它们先手可以必胜。

否则，至少有三个数不是 $k$ 的倍数。看起来比较复杂。

实际上答案只和 $n$ 的奇偶性有关，用数学归纳法证明。

首先 $n=3$ 的情况下，无论如何都会变成前面的对手必胜的情况，所以必输。

$n=2V+1$ 必输可以推出 $n=2V+2$ 必胜。

我们需要证明 $n=2V+2$ 经过一次操作后，整体不被 $k$ 整除的数的个数不小于 $3$ ，就可以进入对面必输的状态。

如果数组中存在 $k$ 的倍数，将其和其它任意数合并显然满足要求。否则，至少有 $4$ 个数不是 $k$ 的倍数，将它们关于 $k$ 取模，必然有两个数同奇偶，将它们操作后是 $[0,2k)$ 的偶数，不可能是 $k$ 的倍数。所以剩下了至少 $3$ 个数不是 $k$ 的倍数，因此可以用到归纳假设。

$n=2V+2$ 必胜可以推出 $n=2V+3$ 必输。

因为 $n=2V+3$ 时，要么操作一次使得整体只有 $2$ 个非 $k$ 的倍数，要么变成归纳假设的情况，无论如何都是对手赢。所以先手必输。

综上，得证。

时间复杂度为 $\mathcal{O}(n)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    n, k = MII()
    nums = LII()
    
    c = 0
    for x in nums:
        if x % k: c += 1
    
    if c == 0:
        print('Rami')
    elif c == 2:
        print('Oussama')
    else:
        print('Oussama' if n % 2 == 0 else 'Rami')
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n, k;
	cin >> n >> k;

	int c = 0;
	for (int i = 0; i < n; i ++) {
		int x;
		cin >> x;
		if (x % k) c ++;
	}

	if (c == 0) cout << "Rami";
	else if (c == 2) cout << "Oussama";
	else cout << ((n & 1) ? "Rami" : "Oussama");

	return 0;
}
```
