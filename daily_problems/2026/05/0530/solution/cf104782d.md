**提示 1：** 要求的是什么？实际上重要的是什么？

如果没啥想法，你可以尝试写一个暴力的做法：

```
x1 = 0
x2 = 0

for i in l ... r:
	x1 = x1 + A[i]
	x2 = max(x2, x1) + B[i]

ans = x2 - x1
```

所以 `x2` 最终数值实际上是什么呢？最终数值是取了最后一次 `max(x2, x1)` 等于 `x1` 的位置 $i$ ，答案是 $\sum\limits_{x=l}^i A_i+\sum\limits_{x=i}^r B_i$ 。

事实上，这也是 $\sum\limits_{x=l}^V A_i+\sum\limits_{x=V}^r B_i$ 对于不同的 $V$ 的最大值。否则最后一次更新的位置不在 $i$ 。

我们再减去 $\sum A_x$ ，发现要求的就是 $B_V+\sum\limits_{x=V+1}^r (B_x-A_x)$ 关于不同 $V$ 的最大值。

这个用线段树很容易维护，只需维护区间和和区间内该数值的最大值即可。

时间复杂度为 $\mathcal{O}(n+q\log n)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    n = II()
    v1 = LII()
    v2 = LII()
    
    vals = [(v2[i] - v1[i], v2[i]) for i in range(n)]
    
    def f(a, b):
        xa, ya = a
        xb, yb = b
        
        x = xa + xb
        y = fmax(ya + xb, yb)
        
        return (x, y)
    
    inf = 10 ** 15
    seg = SegTree(f, (0, -inf), vals)
    
    q = II()
    outs = []
    
    for _ in range(q):
        l, r = GMI()
        outs.append(seg.prod(l, r + 1)[1])
    
    print('\n'.join(map(str, outs)))
```

C++ 做法如下——

```cpp []
typedef pair<long long, long long> node;

long long inf = 1e15;

node op(node a, node b) {
	auto [xa, ya] = a;
	auto [xb, yb] = b;
	return {xa + xb, max(ya + xb, yb)};
}

node e() {return {0, -inf};}

int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int n;
	cin >> n;

	vector<int> v1(n), v2(n);
	for (auto &x: v1) cin >> x;
	for (auto &x: v2) cin >> x;

	vector<node> vec;
	for (int i = 0; i < n; i ++)
		vec.emplace_back(v2[i] - v1[i], v2[i]);

	atcoder::segtree<node, op, e> seg(vec);

	int q;
	cin >> q;

	while (q --) {
		int l, r;
		cin >> l >> r;
		cout << seg.prod(l - 1, r).second << '\n';
	}

	return 0;
}
```
