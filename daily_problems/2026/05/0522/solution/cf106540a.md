**提示 1：** 不断拆分前缀直到不可拆分为止。

**提示 2：** 每个可构造的字符串的拆分结果唯一。

因为不同的前缀可能拼接出同一个字符串，因此得想办法处理这件事。

如果一个前缀字符串存在其某个前缀和后缀相等，则这个后缀实际上也是原字符串的前缀，我们可以把这个字符串进一步切割。不断进行这样的操作直至无法进行为止。

可以证明，每个可构造的字符串拆分结果唯一。因为如果存在两种不同的彻底拆分结果，即至少有一个切割位置不同，则一定对于某个方案的某个子字符串，其后缀是另一个方案的某个子字符串的前缀，也就是整个字符串的前缀，因此还可以进行进一步切割，与两个方案都是彻底切割矛盾。

于是，我们只需考虑彻底切割后的字符串能有多少种凑 $k$ 的方法。

切割后的字符串没有真后缀等于真前缀，这件事可以用 KMP 等方式判断。

得到对应长度后就是简单背包问题了。

时间复杂度为 $\mathcal{O}(nk)$ 。

### 具体代码如下——

Python 做法如下——

```Python []
def main():
    t = II()
    outs = []
    
    mod = 998244353
    
    for _ in range(t):
        n, k = MII()
        kmp = prep(I())
        
        dp = [0] * (k + 1)
        dp[0] = 1
        
        for i in range(k):
            for j in range(n):
                if kmp[j] == 0 and i + j + 1 <= k:
                    dp[i + j + 1] = (dp[i + j + 1] + dp[i]) % mod
        
        outs.append(dp[k])
    
    print('\n'.join(map(str, outs)))
```

C++ 做法如下——

```cpp []
int main() {
	ios_base::sync_with_stdio(false);
	cin.tie(0);
	cout.tie(0);

	int t, mod = 998244353;
	cin >> t;

	while (t --) {
		int n, k;
		string s;

		cin >> n >> k >> s;
		auto kmp = prep(s);

		vector<int> dp(k + 1, 0);
		dp[0] = 1;

		for (int i = 0; i < k; i ++) {
			for (int j = 0; j < n; j ++) {
				if (!kmp[j] && i + j + 1 <= k) {
					dp[i + j + 1] = (dp[i + j + 1] + dp[i]) % mod;
				}
			}
		}

		cout << dp[k] << '\n';
	}

	return 0;
}
```
